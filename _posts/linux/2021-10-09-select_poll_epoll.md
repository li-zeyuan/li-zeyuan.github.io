select、poll、epoll都是Linux内核提供的IO多路复用机制，主要用于同时监视多个文件描述符的事件。
当有事件发生时再通知用户程序处理。主要区别在于数据结构、查询效率、使用场景。
它们之间的区别如下：

|            | select   | poll   | epoll                                                |
| ---------- | -------- | ------ | ---------------------------------------------------- |
| 数据结构   | 数组     | 链表   | 红黑树：存监听文件描述符<br />就绪链表：存就绪文件描述符 |
| 获取fd方式 | 遍历     | 遍历   | 事件回调                                             |
| 时间复杂度 | O(n)     | O(n)   | O(1)                                                 |
| 数据拷贝   | 4次      | 4次    | 内存映射（mmap）3次                                  |
| 连接数     | 一般1024 | 无限制 | 无限制                                               |
| 原理     | 轮询 | 轮询 | 事件驱动                                             |
